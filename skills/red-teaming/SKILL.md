# red-teaming 对抗审视

目标：把用户点名的标的（方案 / 断言 / 草稿 / 计划）当作待击破对象，找出使其不成立的前提、反例或失效场景。攻击方法自行选择，不设流程。

## 成功条件（可证伪）

产出只有两种合格形态：

- **击破**：列出被击破的前提或断言，每条附证据（来源、实际值、失效场景），可被独立复核。
- **未击破**：交出自带查证链的报告——攻击了哪些前提、拿什么实际值或论据对过、各自为何未倒——每条可被抽查。

“整体看起来不错”“基本合理”属于确认式产出，不合格。

## 边界

- 只攻击用户点名的标的；发现的问题如实陈述，修复方案待用户另行要求。
- 击破与否只以证据裁决——标的属于召唤者本人这一事实，不改变裁决标准。

# brainstorming 发散探索

目标：在评判之前把选项面撑到足够大，避免一上来就锁死单一路径。

## 做法
- 先发散后收敛：候选假设、研究问题、解释角度尽量多列，先不评判优劣。
- 用跨领域类比、反向假设（若结论相反会怎样）、alternative explanations 逼出更多可能。
- 收敛时按可行性、新颖性、证据可得性给候选排序。

"太简单，不需要发散"是反模式——正因为觉得显然，才最容易漏掉更好的路径。

## Gate
发散结束必须转 /grilling 逐项拷问收敛，不直接进入执行。
